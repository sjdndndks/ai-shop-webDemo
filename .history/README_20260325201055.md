# AI Shop Monorepo

一个面向学习和演示的 AI 电商 Demo，总仓库同时包含：

- 前端：React + Vite + TypeScript
- 后端：Express + Prisma
- 数据库：PostgreSQL
- AI 选品：OpenRouter

项目目标不是做一个完整电商平台，而是把一个现代 Web 产品最核心的几条链路串起来：

- AI 推荐商品
- 商品加入购物车
- 用户注册 / 登录
- 账户资料、地址、支付方式管理
- 只结算购物车中选中的商品
- 创建订单并落库

## 本地启动

### 1. 启动数据库

```bash
cd ai-shop-server
docker compose up -d postgres
```

### 2. 安装依赖

前端：

```bash
cd ai-shop-assistant
npm install
```

后端：

```bash
cd ai-shop-server
npm install
```

### 3. 初始化数据库

```bash
cd ai-shop-server
npm run db:push
npm run db:seed
```

### 4. 启动后端

```bash
cd ai-shop-server
npm run dev
```

默认地址：

- [http://localhost:3000](http://localhost:3000)

### 5. 启动前端

```bash
cd ai-shop-assistant
npm run dev
```

默认地址：

- [http://localhost:5173](http://localhost:5173)

## 核心业务流

### 1. AI 推荐流

用户输入消息 -> 前端 chat store -> 后端 `/api/chat` -> 本地商品库筛候选 -> AI 选商品 ID -> 返回前端展示。

### 2. 加购流

商品卡片 -> cart store -> 购物车页展示。

### 3. 部分结算流

购物车勾选商品 -> 结算页读取选中项 -> 读取已保存地址 / 支付方式 -> 调用 `/api/orders` 创建订单。

### 4. 账户中心流

设置页 -> `/api/account/*` -> PostgreSQL 持久化 -> 结算页和侧边栏复用。

## 说明

这个项目目前更适合学习、演示和继续迭代，不建议直接以当前形态上线生产。

原因包括：

- 支付还是 mock
- 头像目前保存为 data URL，适合开发期，不适合生产
- 订单能力还不完整
- 缺少正式的权限、监控、审计和风控体系
